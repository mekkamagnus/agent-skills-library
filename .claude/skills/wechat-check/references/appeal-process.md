# WeChat Mini Program Appeal Process

**Source:** WeChat Developer Documentation & Help Center
**Last Accessed:** 2025-02-28
**Language:** English

## WeChat Mini Program Suspension Appeal Guide

---

## Understanding Suspension

### Types of Suspension

| Type | Duration | Cause |
|------|----------|-------|
| Temporary suspension | 1-7 days | Minor violations |
| Service suspension | 7-30 days | Moderate violations |
| Permanent ban | Indefinite | Severe violations |
| Submission rejection | N/A | Application rejected |

### Common Suspension Reasons

1. **Category Mismatch**
   - App content doesn't match declared category
   - Using wrong category (e.g., Education for fitness)

2. **Content Violations**
   - Prohibited content
   - Insufficient content
   - Misleading content

3. **Functional Issues**
   - Excessive simplicity (single page)
   - Bugs or crashes
   - Not functioning as described

4. **Policy Violations**
   - Inducement to share
   - Excessive advertising
   - Hidden paid features

## Appeal Process

### Step 1: Understand the Suspension Notice

**Information provided:**
- Violation reference number
- Specific reason for suspension
- Evidence (screenshots/logs)
- Suspension duration

**Actions:**
- Read carefully
- Identify exact violation
- Note the evidence provided

### Step 2: Assess Your Situation

| Question | Why It Matters |
|----------|----------------|
| Is the category correct? | Determines if reclassification needed |
| Is content prohibited? | May require content removal |
| Is functionality complete? | May need feature additions |
| Is this a mistake? | Appeal with evidence |

### Step 3: Fix the Issues

**Category Issues:**
- Request category change
- Provide explanation of correct classification
- Use Sports/Fitness for fitness platforms

**Content Issues:**
- Remove prohibited content
- Add required pages (Privacy, Terms, Contact)
- Improve content quality

**Functional Issues:**
- Fix bugs
- Add features
- Test thoroughly

### Step 4: Submit Appeal

**Through WeChat MP Backend:**
1. Navigate to "Mini Program" → "Version Management"
2. Find suspended version
3. Click "Appeal" (申诉)
4. Fill appeal form:
   - Appeal reason (申诉理由)
   - Explanation of fixes (整改说明)
   - Supporting evidence (proof of fixes)
5. Submit

**Appeal Template:**
```
申诉理由：Appeal Reason

我们已收到贵平台的 suspension notice，理解关于 [violation reason] 的 concerns。

整改说明：Corrective Actions

1. [Action taken 1]
2. [Action taken 2]
3. [Additional context if applicable]

相关证明：Supporting Evidence

- [Evidence 1]
- [Evidence 2]

我们承诺遵守 WeChat Mini Program 平台规则，望贵平台重新审核。
```

### Step 5: Wait for Review

| Timeline | What Happens |
|----------|--------------|
| 1-3 days | Initial review of appeal |
| 3-7 days | Full re-review (if needed) |
| 7+ days | Complex cases, may require additional info |

**During review:**
- Do not resubmit duplicate versions
- Wait for official response
- Check email/notifications

## Successful Appeal Tips

### 1. Be Specific
- Reference exact violation number
- Explain specific fixes made
- Provide before/after comparisons

### 2. Be Respectful
- Use professional tone
- Acknowledge platform rules
- Show willingness to comply

### 3. Provide Evidence
- Screenshots of fixes
- Test results
- Updated content

### 4. Correct Category
For fitness platforms:
```
本小程序为健身练习平台，提供体育锻炼指导。
This is a fitness practice platform providing physical exercise guidance.
请求归类为：体育健身
Requested category: Sports/Fitness
```

## Common Scenarios & Solutions

### Scenario 1: "类别不符" (Category Mismatch)

**Solution:**
```
We believe the correct category should be Sports/Fitness.
Our platform provides fitness training videos and workout guides.
We do not provide academic tutoring or K-12 education services.
```

### Scenario 2: "内容不足" (Insufficient Content)

**Solution:**
```
We have added the following pages:
- Privacy Policy (隐私政策)
- Terms of Service (服务条款)
- About Us (关于我们)
- FAQ (常见问题)
Screenshots attached showing new content.
```

### Scenario 3: "诱导分享" (Inducement to Share)

**Solution:**
```
We have removed all share prompts and incentives.
Updated screenshots attached showing no share buttons or prompts.
```

## Prevention Best Practices

### Before Submission
- [ ] Correct category selected
- [ ] Privacy policy accessible
- [ ] Terms of service accessible
- [ ] Contact/About page present
- [ ] No prohibited content
- [ ] No inducement to share
- [ ] Functionality complete
- [ ] Tested on real devices

### Ongoing Compliance
- [ ] Regular content review
- [ ] Monitor policy updates
- [ ] Keep documentation current
- [ ] Test after updates
